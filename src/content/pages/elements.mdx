### 博客文章信息行的填写注意事项

对于一篇完整的文章，他的信息行如下：

```plaintext
---
title: 我的博客
meta_title: " "
description: "这是一篇我的博客"
date: 2025-12-28
image: /images/posts/cover.jpg
categories: ["博客","blog"]
authors: ["zzx","wzm"]
tags: ["tag1", "tag2"]
draft: false
---
```

注意事项：

1. `date`和`draft`字段<mark>**不能**加引号</mark>，`title`字段可加可不加

2. date字段的格式必须是YYYY-MM-DD,后面可以加时间，例如`2025-12-28T05:00:00Z`，表示UTC时间的2025年12月28日凌晨5点。如果不加时间，默认是当天的00:00:00时间。
3. `categories`, `authors`, `tags`字段的值是一个数组，即使只有一个元素也需要用`[]`括起来。并且且数组内的元素**需要加引号**，例如:

    ```markdown
    ["分类1", "分类2"] ## 正确
    ['分类1', '分类2'] # 错误
    [分类1, 分类2]     # 错误
    []              # 正确
    [“分类1”，"分类2"]# 错误，注意引号和逗号必须是英文
    ```

    `authors`字段用于表示文章的作者，可以有多个作者，填写时注意名字要和`src/content/authors/`文件夹下的**作者文件名**一致（不带扩展名）。例如`authors: ["wzm", "zzx"]`表示这篇文章的作者是`src/content/authors/wzm.md`和`src/content/authors/zzx.md`两个文件中定义的作者。

4. `draft`字段的值只能是`true`或`false`，不能加引号。如果是`true`，表示这篇文章是草稿，不会被发布。
5. `description`字段用于文章的简短描述，需要加引号，建议填写，这是没打开文章时的预览文本。
6. `image`字段用于文章的封面图片。图片路径是**相对于** `public`文件夹的路径，例如`/images/posts/cover.jpg`表示图片在`public/images/posts/cover.jpg`。<mark>一般把封面图片放在</mark>`public/images/posts/`文件夹下。
7. 每个人写文章的时候注意一下`categories`和`tags`里内容的复用率，避免出现有100个几乎一样的category然后每个里面只有一篇文章的情况。

### 作者文件信息行的填写注意事项

对于每个作者文件，位于`src/content/authors/`文件夹下，例如`src/content/authors/zzx.md`

其中`.md`文件的文件名（不带扩展名）就是作者的标识符，例如`zzx`。

```plaintext
---
title: 作者名字
meta_title: ""
image: 
description: 
social:
    qq: 
    wechat: 
    bilibili:
---
```

注意事项：

1. `title`字段用于表示作者的名字，建议和作者文件名一致。

2. `image`字段用于表示作者的头像图片。图片路径是相对于`public`文件夹的路径，例如`/images/authors/zzx.jpg`。建议放在`public/images/authors/`文件夹下，并且图片命名和作者文件名一致，例如`zzx.jpg`。

3. qq和wechat字段用于表示作者的QQ和微信联系方式，建议不要填qq号，这个仓库是公开的，虽然说被爆破的概率极低，但还是别填，免得给你盗了。想填的话填bilibili主页链接也是可以的。
4. social字段里面不填的部分不能留空，建议填成空字符串`""`，否则会报错。

### 标题的表示

```plaintext

#### Heading 4
```

# 一级标题

### 强调

1. 你来这里是为了 **特别** 的事情还是只是随便看看

    黑体加粗的方式如下，效果如上:

    ```plaintext
    你来这里是为了 **特别** 的事情还是只是随便看看
    ```

2. 你来这里是为了 <ins>特别</ins> /<u>特别</u>的事情

    下划线（html风格）的方式如下，效果如上:

    ```html
    你来这里是为了 <ins>特别</ins> 的事情

    或

    你来这里是为了 <u>特别</u> 的事情
    ```

3. _你来这里_

    斜体的方式（markdown风格）如下，效果如上:

    ```plaintext
    _你来这里_
    ```

4. ~~你来这里~~

    删除线的方式如下，效果如上，注意是英文波浪线喵

    ```plaintext
    ~~你来这里~~
    ```

5. 你来这里是为了 <mark>特别</mark> 的事情

    黄色高亮的方式（html风格）如下，效果如上:

    ```html
    你来这里是为了 <mark>特别</mark> 的事情
    ```

### 表格

| #   |    名     |     姓     |       昵称 |
| :-- | :----------: | :----------: | -----------: |
| 1   | 行:1 列:1 | 行:1 列:2 | 行:1 列:3 |
| 2   | 行:2 列:1 | 行:2 列:2 | 行:2 列:3 |
| 3   | 行:3 列:1 | 行:3 列:2 | 行:3 列:3 |

对应的代码如下：

```markdown
| #   |    名     |     姓     |       昵称 |
| :-- | :----------: | :----------: | -----------: |  #对齐方式在这一行声明
| 1   | 行:1 列:1 | 行:1 列:2 | 行:1 列:3 |
| 2   | 行:2 列:1 | 行:2 列:2 | 行:2 列:3 |
| 3   | 行:3 列:1 | 行:3 列:2 | 行:3 列:3 |

```

`|` 用来分隔列。注意表格的每一行都需要以`|`开头和结尾。对于最后一列，不加`|`也可以，但建议加上以保持一致性。

在第二行声明一列的对其方式`:-- `左对齐，`:--:` 居中对齐，`--:` 右对齐。注意冒号是<mark>英文</mark>冒号`:`，中间的`-`数量没有限制，但必须至少有一个。
